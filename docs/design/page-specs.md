# 页面规格

以下规格对齐当前路由与视图，并标明相对现网的改动意图。布局图为文字线框，便于评审与实现对照。

## 0. 信息架构（现状）

```text
登录
 └─ /reader                  阅读器（文档列表 + TOC + 正文）
 └─ /admin                   管理台（桌面 ≥761px）
      ├─ /documents          文档列表
      ├─ /documents/:id      文档详情 / 版本生命周期
      ├─ /imports            导入中心
      └─ /versions/:id/edit  版本编辑器（含独立预览窗）
```

---

## 1. 登录页 `App.vue`

### 目标

建立品牌识别与「继续阅读」预期，操作仍保持极简。

### 线框

```text
┌──────────────────────────────────────────────┐
│  浅色纸纹/微渐变底（冷灰绿，非紫、非奶油海报） │
│                                              │
│           ┌────────────────────┐             │
│           │ [品牌标]           │             │
│           │ Interview Reader   │             │
│           │ 结构化面试讲义…    │             │
│           │                    │             │
│           │ 用户名             │             │
│           │ 密码               │             │
│           │ [    进入阅读    ] │             │
│           └────────────────────┘             │
│                                              │
└──────────────────────────────────────────────┘
```

### 规格要点

| 项 | 要求 |
|---|---|
| 背景 | `#edf2f4` → 可加极淡径向青绿光晕（透明度 ≤ 6%），忌大图插画抢戏 |
| 卡片 | 单层阴影、8px 圆角、最大宽 400px |
| 主按钮 | 全宽、品牌色、文案「进入阅读」或「登录」 |
| 品牌 | 品牌标 + 产品名同级展示；副文一行说清价值 |
| 动效 | 卡片进入时轻微上移 + fade（可选，150ms） |

## 2. 阅读器 `ReaderView.vue`（核心）

### 2.1 桌面布局（目标态）

系统方案为三栏；现网为两栏。首轮视觉改造可先做「两栏精修」，右侧栏作为增强阶段。

**首轮（两栏精修）**

```text
┌─ Header 64px ─────────────────────────────────────────────┐
│ [≡可选] 文档标题 / 章节副标     [搜索] [舒适度] [主题] … │
├──────────┬────────────────────────────────────────────────┤
│ 文档列表 │                                                │
│ ──────── │              正文柱 720–760px                   │
│ TOC 树   │              章节标题                           │
│          │              段落 / 代码 / 表格 / 提示块        │
│ 280px    │              [上一章]            [下一章]       │
└──────────┴────────────────────────────────────────────────┘
```

**增强（三栏，对接笔记/掌握度 API）**

```text
│ TOC 280 │ 正文 720 │ 笔记/掌握度 280（可折叠） │
```

### 2.2 相对现网的关键优化

| 问题 | 优化 |
|---|---|
| 桌面缺少搜索入口 | Header 增加「搜索」按钮，打开现有搜索抽屉/面板 |
| 正文过宽（900px） | 默认 `min(100%, 740px)`，舒适度可调 640 / 740 / 860 |
| 主题仅明暗 | 增加「护眼」；控件用分段切换而非仅一个按钮 |
| 舒适度缺失 | 弹层：字号、行高、栏宽；本地持久化 |
| TOC 选中与文档选中 | 统一 `--brand-100` 底 + `--brand-700` 字 |
| 进度感知弱 | Header 下细进度条（桌面也可显示文档进度，不只移动端章节条） |
| 代码块 | 顶栏语言标签小写规范；复制按钮方角 6px，hover 用品牌浅底 |

### 2.3 移动端

```text
┌─ sticky header ──────────────┐
│ 菜单  标题…     主题         │
│ ▓▓▓▓▓░░░░ 进度               │
├──────────────────────────────┤
│ 正文单列 padding 18px        │
│ …                            │
│ 底部分页按钮                 │
│                    (FAB)     │
│              目录 / 搜索     │
└──────────────────────────────┘
```

增强（对齐系统方案）：底部工具条「目录 · 上一题 · 标记 · 字体 · 下一题」可逐步替代纯 FAB，降低发现成本。首轮可保留 FAB，但保证搜索与目录双入口文案清晰。

### 2.4 内容块呈现

| 块类型 | 展示要求 |
|---|---|
| 段落 | 预换行保留；段间距 ≈ 0.85em |
| 列表 | 左缩进稳定；题干列表项可加左侧细品牌条（可选） |
| 代码 | 横向滚动；复制成功用简短 toast |
| 表格 | 外层横向滚动；表头弱底 `--surface-2` |
| 提示/引用 | 左边 3px 品牌条 + 浅底（现有 callout 保留） |
| 公式 | 规划 KaTeX；过渡期等宽展示并标注「公式」 |
| 图片 | 过渡期占位改为「图片：{alt}」+ 虚线框，避免空白块 |

### 2.5 复习模式（增强，规格预留）

- 问题块默认折叠答案；点击展开。
- 块旁三态：会 / 模糊 / 不会（色：品牌 / 警告 / 危险浅底）。
- 不改变主阅读栏宽度；状态收入右侧栏或块内轻控件。

## 3. 管理台布局 `AdminLayout.vue`

### 线框

```text
┌────────┬────────────────────────────────────┐
│ Brand  │  页面标题 + 一句说明                │
│ 导航   │  主内容（表格 / 表单 / 双栏）       │
│ ·文档  │                                    │
│ ·导入  │                                    │
│ 折叠   │                                    │
│ 退出   │                                    │
└────────┴────────────────────────────────────┘
```

### 优化点

- 激活项改为青绿底，去掉亮蓝阴影。
- `eyebrow` 可保留，但颜色用 `--brand-500`，字距略收，避免「运营站点」感。
- 主区分页标题层级：Eyebrow（可选）→ H1 → 灰色说明一行。
- 移动端仍跳转阅读器；可在阅读器「关于/菜单」中说明「管理请在桌面打开」。

## 4. 文档列表 `AdminDocumentsView.vue`

```text
标题：文档管理
说明：维护已导入的面试讲义与发布状态

[搜索框........................] [刷新]

┌ 表格 ─────────────────────────────────────┐
│ 标题 / 标签 / 状态 / 更新时间 / 操作      │
└───────────────────────────────────────────┘
```

### 舒适性

- 行悬停浅底；整行可点进详情。
- 状态 Tag：已发布 = 品牌色 plain；草稿 = info；下架 = info dark；勿五彩斑斓。
- 空状态：简短文案 +「去导入中心」主按钮。

## 5. 文档详情 `AdminDocumentDetailView.vue`

现有「当前版本面板 + 历史列表」结构良好，视觉收敛即可：

- 当前版本面板：左侧 4px 品牌 inset（已有）保留；标题数字层级清晰。
- 操作按钮分组：**主路径**（编辑草稿 / 发布）靠右前部；危险（永久删除）分隔或放入更多菜单。
- 历史行：最新草稿弱高亮改为品牌浅渐变，替代蓝色渐变。

## 6. 导入中心 `ImportCenterView.vue`

叙事：「上传讲义 → 流水线处理 → 确认后入库」。

```text
┌ 左：上传与配置 ──────┐ ┌ 右：任务进度 ──────────┐
│ 文件类型 / 选择文件  │ │ 阶段轨 ●─●─●─○─○     │
│ 识别结果条           │ │ 状态大字 + Tag         │
│ 元数据（就绪后）     │ │ 问题列表（可读）       │
│ [提交为草稿]         │ │ 冲突解决（如有）       │
└──────────────────────┘ └────────────────────────┘
```

### 优化点

- 阶段轨文案中文化且固定五步，完成态用品牌色（已有逻辑可保留，统一 token）。
- 问题 `el-alert` 改为紧凑 issue 列表：级别图标 + 一句说明 + 定位（章节/块）若有。
- 主 CTA 仅一个：「提交为草稿」；轮询中按钮 disabled 并显示「处理中…」。
- 上传区图标与虚线框使用品牌色，避免 Element 默认蓝。

## 7. 版本编辑器 `VersionEditorView.vue`

### 目标

降低「工具条过载」感，同时保留树 + 块编辑 + 预览能力。

```text
┌ Header：返回 | 文档名 vN | 未保存提示 | [预览] [保存] [更多▾] ┐
├──── 树 272px ────┬──────── 内容编辑 ─────────────────────────┤
│ 搜索节点         │ 节点标题 / 属性（可折叠）                 │
│ TOC 树           │ 块列表 + 当前块表单                       │
│                  │                                           │
│                  │  （浮层预览可拖拽，默认贴右）               │
└──────────────────┴───────────────────────────────────────────┘
```

### 舒适性规则

1. Header 只保留 2 个主按钮：保存、预览开关；其余进「更多」。
2. 节点属性默认折叠，减少与块编辑抢视线。
3. 未保存状态用品牌点 +「未保存」文字，不只靠按钮 disable。
4. 预览窗背景与阅读器浅色主题一致，预览即「读者所见」。
5. 窄屏（&lt;1050px）允许树面板变为可调整宽度或抽屉，避免三区挤压。

## 8. 全局反馈与离线

| 场景 | 展示 |
|---|---|
| 离线 | 底部深色条，文案「离线中，进度将稍后同步」 |
| SW 更新 | 品牌色条 +「刷新以更新」 |
| 保存进度失败入队 | 不打断阅读；可在菜单显示「待同步 N 项」 |
| 危险确认 | Element MessageBox；标题说清后果 |

---

## 9. 响应式断点

| 断点 | 行为 |
|---|---|
| ≤760px | 阅读器移动布局；禁止管理台 |
| 761–1049px | 管理台单列；编辑器树可缩 |
| ≥1050px | 管理双栏、编辑器完整工作台 |
| ≥1280px | 阅读器可开右侧增强栏 |

与现有 `shared/responsive.ts` 的 760px 保持一致，避免再引入第三套断点。
