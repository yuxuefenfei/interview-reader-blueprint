1.3 并发与线程池

请说明 ThreadPoolExecutor 的核心参数，以及任务提交后的执行流程。面试中常见追问是：队列满了之后会发生什么？如何避免任务堆积打垮下游？

答题要点
核心线程 → 队列 → 最大线程 → 拒绝策略。回答时先画流程，再落到业务场景中的队列选型。
JAVA 
ThreadPoolExecutor pool = new ThreadPoolExecutor(
  4, 16, 60L, TimeUnit.SECONDS,
  new LinkedBlockingQueue<>(1000),
  new ThreadPoolExecutor.CallerRunsPolicy()
);

拒绝策略要结合调用方是否可阻塞来选。对用户请求链路，CallerRunsPolicy 可以形成天然背压；对异步批处理，更常见的是记录失败并告警，而不是静默丢弃。
