# 产品主题与设计原则

## 1. 产品一句话

**Interview Reader** 把 PDF / Excel / Markdown 面试资料转成结构化 AST，再以可恢复进度、可搜索、可复习的方式碎片化阅读。

界面不是营销落地页，也不是通用 CMS。它是：

> 一份安静、专注、结构清晰的「面试讲义阅读工作台」。

## 2. 用户与场景

| 角色 | 主要场景 | 界面重心 |
|---|---|---|
| 备考读者 | 通勤刷题、章节精读、搜关键词、回看进度 | 阅读器：正文可读性、目录定位、搜索可达 |
| 内容管理员 | 导入讲义、修订草稿、发布版本 | 管理台：状态清晰、主操作唯一、减少误触 |
| 导入操作者 | 上传 → 看阶段 → 处理冲突 → 提交草稿 | 导入中心：进度叙事、问题可读、下一步明确 |

## 3. 视觉叙事（Mood）

用「书桌 + 讲义」而非「仪表盘 + 卡片墙」：

| 关键词 | 含义 | 界面落点 |
|---|---|---|
| 讲义感 | 有纸感、章节层级、清晰正文柱 | 阅读区窄而稳，标题层级分明 |
| 结构感 | TOC / 块 / 版本是一等公民 | 左侧导航始终可辨，状态标签克制 |
| 专注 | 减少装饰与竞争信息 | 首屏只做当前任务，工具收入二级入口 |
| 青绿信任 | 延续现有品牌色，表达「整理过的可靠内容」 | 主操作、进度、选中态统一用青绿 |

氛围参考（文字描述，非素材库）：

- 浅色：清晨书桌、略冷的纸白、青绿墨水标注。
- 深色：夜间台灯下的护眼阅读，表面略抬升，强调正文而非霓虹。
- **不做**：紫蓝渐变 SaaS、暖奶油衬线海报风、报纸栏排版、大面积发光描边。

## 4. 品牌表达

| 元素 | 规范 |
|---|---|
| 产品名 | `Interview Reader`（中文场景可副标「面试阅读器」） |
| 简称 | `Reader`（PWA short_name 可保留） |
| 标志 | 打开的讲义图标：青绿色渐变底 + 白色书页轮廓 + 三条题干线；登录页、管理侧栏、浏览器与 PWA 共用同一 `icon.svg` |
| 品牌位 | 登录页与侧栏品牌区是首要识别；阅读器顶栏以**当前文档标题**为主，产品名次之 |
| 语气 | 短句、操作导向：「继续阅读」「提交为草稿」「发布此版本」 |

登录页建议文案：

- 标题：`Interview Reader`
- 副文：`结构化面试讲义，随时从上次读到的地方继续。`
- 避免空泛口号（如「智能驱动未来」）。

## 5. 设计原则

1. **阅读优先**：读者端任何视觉决策以长文舒适度为先；装饰不得抢正文。
2. **一屏一事**：每个视图一个主任务；次要能力进抽屉、菜单或右侧可折叠栏。
3. **结构可见**：目录深度、版本状态、导入阶段必须一眼可读，不靠悬停才懂。
4. **操作舒适**：触控 ≥ 44px；危险操作二次确认；主按钮唯一且位置稳定。
5. **主题连贯**：登录 / 阅读 / 管理共用同一套 token；管理台可更深色侧栏，但主色仍是青绿，不用第二套「亮蓝运营风」抢戏。
6. **渐进增强**：先优化已有页面观感与可达性；笔记 / 掌握度 / 导出等按系统方案分阶段接入，不阻塞首轮视觉统一。

## 6. 明确反模式

| 反模式 | 原因 |
|---|---|
| 首屏堆统计条、标签云、多卡片 | 破坏阅读工作台专注感 |
| 正文区套厚重卡片阴影 | 增加视觉噪音，像后台表单 |
| 管理侧栏用亮蓝选中、阅读用青绿 | 双品牌色分裂 |
| FAB 成为桌面主入口 | 桌面应有明确工具栏；FAB 仅移动端 |
| 用 emoji 当状态图标 | 与专业讲义气质不符 |
| 默认强制深色 | 深色是可选主题，不是默认品牌 |

## 7. 成功标准（体验）

- 打开已发布文档 3 秒内能识别「在读哪一章」。
- 桌面端无需摸索即可发起章内搜索。
- 连续阅读 30 分钟不产生明显眼疲劳（字号 / 行高 / 对比度达标）。
- 管理员完成「导入 → 处理问题 → 提交草稿」路径时，始终知道当前阶段与下一步。
- 换肤（浅 / 深 / 护眼）后布局不跳动，仅色值变化。
